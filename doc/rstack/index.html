Thread stack traces of remote processes.

rstack (named after Java's jstack) uses libunwind's ptrace interface to capture stack traces of the threads of a remote process. It currently only supports Linux with a kernel version of 3.4 or higher, and requires that the /proc pseudo-filesystem be mounted and accessible.
